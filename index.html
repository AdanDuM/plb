Overview


Philosophy1) Benchmark language implementations, not individual programs (simple tasks with few pitfalls).
2) Benchmark one language a time, not a mixture of languages (no libraries in foreign languages especially in C).
CPUIntel(R) Xeon(R) CPU E5440 @ 2.83GHz
Memory16GB
OSDebian GNU/Linux 5.0
Source codehttps://github.com/attractivechaos/langbench

Benchmarks


matmulCubic-time matrix multiplication of two 1000x1000 matrics. This benchmark evaluates the performance of nested loops with a simple inner loop, which is frequent in scientific computing.

Notes


General 1) C programs are compiled with "gcc/clang -O3 -fomit-frame-pointer" or "icc -O3 -fomit-frame-pointer -xSSE4.1"
2) Due a technical complication, Java programs are compiled on Mac using 1.6.0_24 and run on Linux with 1.6.0_14.
3) Mono-sgen is used for implementations requiring the .NET framework. Mono-sgen is usually faster but costs more memory than mono.
matmul 1) The built-in "Matrix" class in Ruby does not transpose the second matrix before multiplication. Using the built-in class is twice as slow.
2) Testing R in this benchmark is actually unfair to other scripting languages.
